# Описание файлов проекта

## dwh_lol_data.sql
SQL-скрипт для создания хранилища данных (DWH) League of Legends. Содержит:

- **Таблицы измерений:**
  - `dim_player` – игроки (puuid, leaguePoints, rank, wins, losses, статусные флаги).
  - `dim_champion` – чемпионы (champion_id, champion_name, title, tags).
- **Таблицу фактов:**
  - `fact_match_participant` – участники матчей (match_id, puuid, champion_id, статистика, kda, внешние ключи на измерения).
- **Ограничения целостности:** внешние ключи, уникальность пары `(match_id, puuid)`.
- **Заполнение таблиц** данными из исходных CSV (`players`, `champions`, `matches`) с защитой от дублей и дополнением недостающих записей.

## lol_data.db
Файл базы данных SQLite, создаваемый при выполнении `dwh_lol_data.sql`. Содержит все данные в нормализованном виде, готовые для аналитических запросов. Является основой для дашбордов и отчётов.

## lol_dwh_analysis.sql
Набор аналитических SQL-запросов для полученного хранилища. Включает:

1. Топ‑15 чемпионов по винрейту (минимум 5 матчей).
2. Среднее количество побед у топ‑15 игроков по количеству матчей (сортировка по числу матчей).
3. Топ‑15 чемпионов с наибольшим количеством поражений (минимум 5 матчей, с процентом поражений и долей в общих поражениях).
4. Среднее количество побед/поражений по всем игрокам и их соотношение (win/loss ratio).
5. Распределение игроков по очкам лиги (LP) с группировкой по интервалам 200 LP и указанием чемпионов, на которых они играли.
6. Распределение длительности матчей (квартили, среднее, минимум, максимум) с использованием `NTILE(4)`.
7. Среднее количество убийств vs смертей по чемпионам (минимум 5 матчей, сортировка по убыванию средних убийств).

Все запросы используют связи между `fact_match_participant`, `dim_player` и `dim_champion`.
